16 de abril de 2018

FEATURE:

Algoritmos de hiperparâmetros: ao ajustar os hiperparâmetros no job de treinamento, agora é possível especificar um algoritmo de pesquisa em HyperparameterSpec . Veja a seguir os valores disponíveis:

  • GRID_SEARCH : uma pesquisa de grade simples dentro do espaço viável. Essa opção será útil principalmente se você quiser especificar um número de tentativas maior do que o número de pontos no espaço viável. Nesse caso, se você não especificar uma pesquisa de grade, o algoritmo padrão do Cloud ML Engine poderá gerar sugestões duplicadas. Para usar a pesquisa de grade, todos os parâmetros precisam ser do tipo INTEGER , CATEGORICAL ou DISCRETE .
  • RANDOM_SEARCH : uma pesquisa aleatória simples dentro do espaço viável.

Se você não especificar um algoritmo, o job usará o algoritmo padrão do Cloud ML Engine, que controla a pesquisa no espaço de parâmetros de maneira mais eficaz para chegar à solução ideal. Para mais informações, consulte a visão geral do ajuste de hiperparâmetros .

v1 (8 de março de 2017)

FEATURE:

Anunciada a disponibilidade geral do AI Platform Training. A versão 1 do Cloud ML Engine está disponível para uso geral em modelos de treinamento e de implantação e geração de previsões em lote. O recurso de ajuste de hiperparâmetro também está disponível para uso geral, mas a previsão on-line e as máquinas habilitadas para GPU permanecem na versão Beta.

FEATURE:

A previsão on-line está agora na fase de lançamento Beta. No momento, o uso dela está sujeito à política de preços do Cloud ML Engine e à mesma fórmula de preços aplicada à previsão em lote. Enquanto estiver em Beta, a previsão on-line não se destinará ao uso em aplicativos críticos.

CHANGED:

Os ambientes que o Cloud ML Engine usa para treinar modelos e conseguir previsões foram definidos como versões de ambiente de execução do Cloud ML Engine. Especifique uma versão de ambiente de execução compatível para usar ao realizar o treinamento, definir um recurso de modelo ou solicitar previsões em lote. Neste momento, as versões de ambiente de execução diferem principalmente em relação à versão do TensorFlow compatível com cada uma delas, mas outras diferenças poderão surgir ao longo do tempo. Procure os detalhes na lista de versões de ambiente de execução .

CHANGED:

Agora é possível executar jobs de previsão em lote com base nos SavedModels do TensorFlow que estiverem armazenados no Google Cloud Storage e não hospedados como uma versão de modelo no Cloud ML Engine. Quando você criar o job, use o URI do SavedModel em vez de fornecer um modelo ou ID de versão.

DEPRECATED:

O uso do SDK do Google Cloud Machine Learning, anteriormente lançado como Alfa, está suspenso e não é mais aceito desde 7 de maio de 2017. A maior parte da funcionalidade disponibilizada pelo SDK foi transferida para o novo pacote do TensorFlow, tf.Transform (em inglês). É possível usar a tecnologia ou ferramenta que você preferir para fazer o pré-processamento dos dados de entrada. No entanto, recomendamos o tf.Transform e os serviços disponíveis no Google Cloud Platform, incluindo o Google Cloud Dataflow, o Google Cloud Dataproc e o Google BigQuery.
